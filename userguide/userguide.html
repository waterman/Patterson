Quickstart

Structure of resources

To generate a website using Patterson you organize the files making up your site in in source directory. The layout of the source directory corresponds to the directory structure of the finally generated site. Patterson walks along the source directory and copies the files to a target directory. Files which are recognized as templates are evaluated before copied.

The default directory structure for Patterson is as follows: 
    +--+ site/ /*default source folder containing the templates and resources*/
       +--+ myFolder1/
          +-- myFile1.groovy.html
          +-- myImage1.gif  
    +--+ build/ /*default target folder containing the finally generated site*/
       +--+ myFolder1/
          +-- myFile1.html
          +-- myImage1.gif
    +-- build.xml /*optional ant build file, calling the generator*/

Template processing

Patterson assumes, that files in the source directory having a name like <filename>.groovy.<extension> are Groovy templates. These templates are evaluated, the ".groovy" part of the filename is removed and the result of the template is copied to its corresponding folder in the target folder.

Template features

Groovy templates are a mixture of plain target content and Groovy code. The Groovy code typically adds content dynamically. Several helper Objects are available in the context of each template:

Variable nameDescription
generatorThe generator object itself, providing access to the entire generator runtime
templateSupportProviding methods usefull for templates
