Quickstart

Structure of resources

To generate a website using Patterson you organize the files making up your site in a source directory. The layout of the source directory corresponds to the directory structure of the finally generated site. Patterson walks along the source directory and copies the files to a target directory. Files which are recognized as templates are evaluated before copied.

The default directory structure for Patterson is as follows: 
+--+ site/ /*default source folder containing the templates and resources*/
   +--+ myFolder1/
      +-- myFile1.groovy.html
      +-- myImage1.gif  
+--+ build/ /*default target folder containing the finally generated site*/
   +--+ myFolder1/
      +-- myFile1.html
      +-- myImage1.gif
+-- build.xml /*optional ant build file, calling the generator*/

Template processing

Patterson assumes, that files in the source directory having a name like <filename>.groovy.<extension> are Groovy templates. These templates are evaluated, the ".groovy" part of the filename is removed and the result of the template is copied to its corresponding folder in the target folder.

Template features

Groovy templates are a mixture of plain content -for example HTML- and Groovy code. The Groovy code typically adds or omits content dynamically. Please have a look at the Groovy Template documentation for a general information about Groovy and Template.

Patterson provides access to several helper objects in the context of each template:

Variable nameDescription
generatorThe generator object itself, providing access to the entire generator runtime
templateSupportProviding methods useful for templates
bindingA map which carries arbitrary user objects. Use this map to control the dynamic behaviour of your template. The binding map is initialized with the settings of the generator and the settings of the current resource. Each template gets its own binding. You can pass on the binding this binding to make it available to the included templates.

Including other templates

You can use the templateSupport from within your template to include other templates as follows: 
${templateSupport.evalTemplate(binding,'${srcDir}/includes/header.include')}

Calculating the URL to other resources

When including templates like menu bars or headers, the URLs in these fragments usually dependent on the location of the including resource. You can use the templateSupport to calculate the correct relative URL from the current resource to another: 

<img src="${templateSupport.urlTo('${srcDir}/assets/Logo.gif')}"/>

Images galleries

Patterson has build-in support for generating image galleries. Put all the images for the gallery into one folder, along with the corresponding thumbnails. Images and thumbnails are copied unchanged to the target, so it is required for them to have to expected dimensions. Put an additional template with content like below in the same directory to generate a gallery: 

${org.patterson.gallery.GenerateGalleryProcessor.run(generator,binding); }
